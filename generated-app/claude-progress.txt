### Known Limitations
- Infrastructure NOT deployed (no write AWS access, CI/CD not configured in repo)
- Frontend uses mock data fallback (API not available yet)
- VITE_API_URL not set (would be set once stack deploys)
- The mock data fallback means mutations (create, update, delete) don't persist across page reloads

### How to Resume
1. If infrastructure deploys:
   - Get API URL from CloudFormation outputs
   - Create frontend/.env with VITE_API_URL=<api-url>
   - Call POST /api/seed to populate database
   - Frontend will automatically use real API
2. If need to continue development:
   - Run `./init.sh` to start dev server
   - All npm installs are needed (no workspaces, each package has its own node_modules)
