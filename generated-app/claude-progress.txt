## Architecture

### Monorepo Structure
```
generated-app/
├── shared/          - Zod schemas, types, endpoint definitions
├── infrastructure/  - CDK stack (DynamoDB, Lambda, API GW, S3, CloudFront)
├── backend/         - Lambda handler with path-based routing
├── frontend/        - React 18 + Vite 7 + Tailwind CSS v4
└── e2e-tests/       - Playwright test files
```

### Shared Package (@canopy/shared)
- Zod schemas: Project, Issue, Sprint, Epic, Announcement, Comment
- TypeScript types exported via inference
- Endpoint route map
- Build: `tsc` emits to `dist/`

### Infrastructure (CDK)
- DynamoDB single-table design (canopy-ankit-aidlc-testing-canopy-table)
- Lambda Node.js 20 with bundled esbuild (NodejsFunction)
- API Gateway HTTP API with CORS
- S3 + CloudFront for static frontend hosting
- SSM parameter for deploy state
- Tests: 11 passing (snapshot + fine-grained assertions)

### Backend (Lambda)
Path-based router handling all API routes:
- GET/POST /api/projects
- GET /api/projects/:id
- GET/POST /api/projects/:id/issues
- PUT /api/issues/:id (status, priority, assignee updates)
- PUT /api/issues/:id/move (drag-and-drop status changes)
- GET/POST /api/projects/:id/sprints
- PUT /api/sprints/:id
- GET/POST /api/projects/:id/epics
- GET/POST /api/announcements
- PUT/DELETE /api/announcements/:id
- GET/POST /api/issues/:id/comments
- DELETE /api/comments/:id
- POST /api/seed (seed database)

### Frontend
- React 18 + React Router v6 + TanStack Query v5
- Vite 7 dev server with proxy to deployed API
- Tailwind CSS v4 with custom theme
- @dnd-kit for Kanban board drag-and-drop
- Recharts for dashboard charts
